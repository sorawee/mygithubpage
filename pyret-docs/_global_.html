3.1 Global Utilities

3.1.1 Built-in Utility Functions
torepr :: (val :: Any) -> String

Creates a string representation of the value that resembles an expression that could be used to construct it. This is what the REPL and test-results printer use to display values.

Examples:
check:
  # torepr wraps strings in quotes
  torepr("this-will-have-quotes") is "\"this-will-have-quotes\""

  # torepr on lists always prints with [list: ...] notation
  torepr(link(1, empty)) is "[list: 1]"

tostring :: (val :: Any) -> String

Creates a string representation of the value for display, that is value-dependent. Error messages, for example, have different tostring results that print line and column information and a formatted error message, which is much different from their torepr.

raise :: (val :: Any) -> Nothing

Raises the value as an error. This usually stops the program and reports the raised value, but errors can be caught and checked in tests by raises and by check: blocks.

3.1.2 Built-in Types
Any

A type specification that permits all values. This is mainly useful in built-in language forms, like in equality or torepr, which truly do handle any value. Pyret programs that use Any on their own can usually be restructured to not use the annotation and be clearer about what data they are working with.

Number

The type of Numbers.

Boolean

The type of Booleans.

String

The type of Strings.

Table

The type of Tables.

RawArray

The type of RawArray.

Nothing

The type of the special value nothing, used in contexts where the program evaluates but has no meaningful answer by design (see, for example each).

Object

The type of all values constructed from data constructors and singletons, and by object literals.

Function

The type of all function values.

Method

The type of all method values; most Pyret programs should never need to work with method values directly.

3.1.3 Type Predicates

A number of functions are available to tell which kind of builtin value a particular value is.

is-boolean :: (val :: Any)

Checks whether the provided argument is in fact a boolean

Returns true for true and false, and false for all other values.

is-string :: (val :: Any)

Checks whether the provided argument is in fact a string

Returns true for strings, false for non-strings. Strings can be written "text" or 'text', and may not span multiple lines. Allowed escapes are \n (newline), \r (carriage return), \t (tab), \[0-8]{1,3} for octal escapes, \x[0-9a-fA-F]{1,2} for single-byte hexadecimal escapes, or \u[0-9a-fA-F]{1,4} for double-byte Unicode escapes. Additionally, \" escapes a double-quote within a double-quoted string, and \' escapes a single quote within a single-quoted string.

Multi-line string literals may be written ``` text ```. The same escape sequences are valid as for single-line strings. Leading and trailing whitespace of the string are trimmed.

is-number :: (val :: Any)

Checks whether the provided argument is in fact a number

Returns true for numbers, false for non-numbers. Numbers are

  • Integers, e.g. 345 or -321

  • Rationals, e.g. 355/113 or -321/6789

  • Inexact numbers, e.g. 123.4567 or -0.987

  • Complex numbers, e.g. 1+2i, where the real and imaginary components may be integers, rationals or inexact numbers

is-function :: (val :: Any)

Checks whether the provided argument is in fact a function

Returns true for functions, false for non-functions

is-nothing :: (val :: Any)

Checks whether the provided argument is in fact nothing

is-object :: (val :: Any)

Checks whether the provided argument is in fact an object

is-method :: (val :: Any)

Checks whether the provided argument is in fact a method

is-raw-array :: (val :: Any)

Checks whether the provided argument is in fact a raw array
